# AZMusic Windows Server

This package runs the AZMusic backend for family devices on the same network.

## First Run

1. Double-click `Setup AZMusic Server.cmd`.
2. Install or configure Audiveris, MuseScore, and Tesseract when the setup output says they are missing.
3. If Windows asks for firewall access, allow the server on private networks. For automatic firewall setup, run setup/start as Administrator.
4. Double-click `Start AZMusic Server.cmd`.
5. Confirm the start window shows at least one `LAN setup URL`.
6. Double-click `Open Server Setup Page.cmd`.
7. Scan the parent/admin QR code with the parent AZMusic app.

After the parent device is paired, add student devices from the parent section in the app. Student QR codes are generated by the parent device and are tied to the selected student profile.

If pairing times out, open one of the LAN setup URLs from the phone/tablet browser. If it does not load, the device is not on the same network or Windows Firewall is blocking inbound TCP port `8000`.

## Processing Tools

The server package includes helper scripts for opening installer pages or using `winget`, but it does not bundle the full third-party tools. Real processing requires:

- Audiveris for OMR and MusicXML generation.
- MuseScore for rendering edited MusicXML back to review PDFs.
- Tesseract OCR for title, metadata, and book-page text extraction.

Tool paths can be configured from the parent processing settings screen after pairing. Advanced users can also edit `server\.env`.
